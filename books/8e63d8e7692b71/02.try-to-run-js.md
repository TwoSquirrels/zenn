# とりあえず試してみよう！

**プログラミング**、 **JavaScript** 、これらの言葉の意味ははじめはよくわからないと思います。  
プログラミングとは、コンピュータに対して動き方を指定することで、  
JavaScript はプログラミングをするのに使われる言語の一つです。  
と、言葉で説明してもわかりにくいですね。  
**物は試しです**。まずは JavaScript を試してみましょう！  

:::message
JavaScirpt は「**ジャヴァスクリプト**」などと読み、よく「**JS**」と略されます。  
名前の似た Java というプログラミング言語もありますが、これは JavaScript とは**別物**なので注意しましょう。  
「**Java と JavaScript はインドとインドネシアくらい違う**」と覚えておくといいでしょう。  
:::

## どうやって JavaScript を試すの？

**Web ブラウザ** (サイトを見るためのソフト) で **F12 キーを押し**たり **Ctrl+Shift+I を同時押ししたり**すると  
開発者ツール (デベロッパツール) と言う画面が出てくると思います。  
  
![DevTools のスクショ](/images/devtools.png)  
  
ただし、使っているパソコンやブラウザによっては出来ないことがありますので、  
できない場合は Google 検索を使い方法を調べてみましょう。  
  
検索の例 (Mac で Safari ブラウザを使っている場合):  

```
Mac Safari 開発者ツール 開き方
```

開発者ツールが開けたら、「**Console**」を開いてください。  

![DevTools Console のスクショ](/images/devtools_console.png)  

これで JavaScript を試す準備が整いました！  

## 早速ソースコードを書いてみよう！

**プログラム**とは、コンピュータに対して処理して欲しいことの手順を書いた手紙のようなものです。  
またプログラムのテキストのことを「**ソースコード**」や「**コード**」と呼びます。  
これらの言葉は似ているため、混同されて使われることも多いです。  

:::message
プログラムを作ることを「**プログラミング**」といいます。  
:::

それではこの画面に写真と全く同じプログラムを打ち込んでみてください。  

![開発者ツールに alert(""); と書いたスクショ](/images/alert.png)  

ただし、**一語一句まったく同じ文字**で書く必要があります。**すべて半角文字**なことに注意してください。  
書けたら、 Enter キーを押してみてください。すると空っぽのダイアログが表示されるはずです。  

:::message
エラーが出てしまいうまく動かなかった場合は、どこかが間違っていると思います。  
ダブルクオーテーション (`"`) はシングルクオーテーション (`'`) と書き換えることもできますが、  
シングルクオーテーション２つでダブルクオーテーション１つ**ではない** (`"` ≠ `''`) ことに注意しましょう。  
:::

![ダイアログのスクショ](/images/dialog_sample.png)

プログラミングとはこのように**コンピュータに命令すること**なのです。  
例えばさっきのプログラムだと、「**空のダイアログを出せ**」という命令になります。  
プログラミングが何なのか、なんとなくわかりましたでしょうか。  
  
プログラミング、面白いですか？  
...まあ、まだ面白くはないと思います。  
でもプログラミング学習を続けていれば**どんどん面白くなっていく**はずです！

# JavaScript が動く仕組み

## JavaScript はどこで動く？

今回は手軽さ重視で**開発者ツールから JavaScript を実行してみました**が、  
JavaScript は **Web ページ内に埋め込み**ブラウザ上で動かしたり  
**[Node.js (ノードジェーエス)](//nodejs.org/) というソフトウェア上で**動かしたりすることもできます。  

![JS の動く仕組みのイメージ画像](/images/js_moving.drawio.png)  

この本ではほとんど Web ページ内に埋め込んでブラウザ上で動かす形となります。  
プログラムのデバッグ (バグを探したりバグが起きた時にその原因を調べること) をする時に開発者ツールを使うこともあります。  
Node.js はゲーム製作においては、オンラインゲームを作るなら使う機会があるかもしれません (難易度はかなり高いです)。  

## コンピュータの仕組み

どうして JavaScript のプログラムを書いただけでコンピュータはそれを処理できるのでしょうか？  
この仕組みを知るには、まずコンピュータの仕組みについて理解する必要があります。  
  
コンピュータは、「**CPU**」という**プログラムを動かす**コンピュータの**脳みそ**にあたる部品や、  
「**メモリ**」という**一時的にデータを置いておく**コンピュータにとっての**作業机**にあたる部品、  
「**ハードディスク (HDD)**」や「**SSD**」などの**大量のデータをしまっておく**コンピュータにとっての**倉庫**にあたる部品などを  
「**マザーボード**」と呼ばれる**プリント基盤**で繋ぐことで動いています。  

![コンピュータの中身](https://kuina.ch/images/l6prog/basic1/disassembling.png)  
コンピュータの中身 (&copy;Kuina-chan)  

ただし、 CPU はそのまま JavaScript をプログラムとして認識してくれません。

【参考】 `Hello, World!` を出力するマシン語の一部  

:::message
電圧が低い状態を `0` 、高い状態を `1` として表現しているのでコンピュータにとって 2 進数は都合が良いのです。

そこで生み出されたのが「**アセンブリ言語**」という、  
マシン語を 16 進数で表しそれぞれの数字を分かりやすい単語に一対一で対応付けられた言語です。  
「**アセンブラ**」というプログラムを使うことでアセンブリ言語をマシン語に変換することができます。  

```asm
section .data
    msg db "Hello, World!", 0ah

section .text
    global _start
_start:
    mov rax, 1
    mov rdi, 1
    mov rsi, msg
    mov rdx, 13
    syscall
    mov rax, 60
    mov rdi, 0
    syscall
```

↓ アセンブラが変換  

```
...

00000000 00101110 01110011 01101000
...
```

CPU のメーカー (Intel や AMD など) によってマシン語は微妙に異なります。  
マシン語が異なればアセンブリ言語も当然異なります。  
:::

ただ、アセンブリ言語でもかなり難しく書くのが大変でした。  
しかし **FORTRAN** というプログラミング言語の登場によりプログラミングのハードルは大きく下がりました。  
その後、様々な言語ができました。  

## プログラミング言語の種類

プログラミング言語は大きく「コンパイラ型言語」と「インタプリタ型言語」という２種類の言語に分類することができます。  
その種類によって、ソースコードがプログラムとして動く仕組みが違います。  

### コンパイラ型言語

コンパイラ型言語とは、ソースコードを事前にマシン語に変換 (**翻訳**と呼びます) しといて  
実行時はそのマシン語のプログラムを直接動かすだけで済む、という言語のことです。  
FORTRAN や、 C 言語とその派生の C++ 、また最近流行りの Go や Rust などの言語がこれにあたります。  

![コンパイル](https://kuina.ch/images/l6prog/basic2/compilation.png)  
コンパイル (&copy;Kuina-chan)  

コンパイラ型言語をマシン語に変換することを「**コンパイル**」と言い、  
コンパイルをするプログラムを「**コンパイラ**」と言います。  

### インタプリタ型言語

インタプリタ型言語とは、ソースコードを実行時に一文ずつ、「**中間言語**」と言うプログラムが読み取りやすい形式に翻訳しながら実行する言語のことです。  
JavaScript や Ruby 、また最近流行りの Python などの言語がこれにあたります。  
ただし、 JavaScript は実はコンパイラ型言語でもあります。これについては後で詳しく解説します。  

![インタプリタ](/images/interpreter.drawio.png)  
インタプリタ  

インタプリタ型言語はコンパイラ型言語と比べ実行時に翻訳するため実行速度は遅いですが、  
新しいコードをコンパイルせずにそのまま実行できるというメリットがあります。  
ただし、 CPU はインタプリタ型言語のソースコードをそのままプログラムとして動かすことはできないので、  
インタプリタ型言語はその言語を翻訳し動かすためのプログラムが別で必要となります。  
このようなプログラムを「**インタプリタ**」と言います。  
他にもそれぞれにメリットやデメリットはありますが、気になる方は調べてみてくだだい。  

:::message
「インタプリタ型言語」はよく「スクリプト言語」と混同して使われますが、意味はまったく違います。  
どう違うのかは、自分で調べてみてください。  
:::

### JavaScript はコンパイラ型言語でもある！？

JavaScript は手動でコンパイルをせずにそのまま実行できるので、一見するとインタプリタ型言語のように思えます。  
JavaScript は「V8」や「SpiderMonkey」などのプログラムによって動きます。  
これらのプログラムを「**JavaScript エンジン**」と呼び、「**JIT コンパイラ**」を持っています。  
JIT コンパイラとは Just-In-Time コンパイラの略で、プログラムの実行時にコンパイルをしてから実行するというものです。  
ここでは説明するのが難しいので、インタプリタ型言語とコンパイラ型言語の中間、と理解しといてください。

ブラウザや Node.js はそれぞれ JavaScript エンジンを内蔵しているため、  
JavaScript を動かすことができるのです。  
例えば、 Google Chrome ブラウザや Node.js には V8 エンジンが使われていて、  
Firefox ブラウザには SpiderMonkey が使われています。  

:::message
Java や C# などの言語も JIT コンパイラを採用していますが、  
JavaScript と違ってこれらは手動で中間言語にコンパイルしとく必要があります。  
ただしその分実行時にコンパイルをしないためプログラムをすぐに動かすことができます。  
またコンパイラ言語とも違い、コンパイル後のファイルがマシン語ではないため CPU の種類が違くても動かすことができます。  
:::
