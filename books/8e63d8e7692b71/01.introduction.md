## Web ゲーム制作の理由

どんぐり「この本はなんでプログラミング入門を Web ゲーム制作でやるの？」

りすりす「まず、ゲーム制作の理由は、楽しいから！」

どんぐり「確かに楽しい方がいいね」

りすりす「**ゲーム制作だとモチベを維持しやすい** んだよね」

どんぐり「ゲームは身近なものだし、そのものが面白いから？」

りすりす「うん。だからゲーム制作はプログラミングを好きになる近道だと思うんだ」  
りすりす「もちろん個人差はあると思うんだけど、頑張ってプログラミングをしてできたものが何の役にも立たなくて面白くないものだったら」  
りすりす「飽きちゃったり挫折しちゃう人が多いんじゃないかなって」

どんぐり「なるほど！」

りすりす「**Web ゲーム制作なら Web ブラウザ上だけでもできる** からだよ」

どんぐり「なるほどねー」

## わからないことがあったとき

りすりす「これはプログラミングに限った話じゃないと思うんだけど」  
りすりす「**わからないことがあったときは、まず自分で調べる習慣をつけよう**」　　
りすりす「例えば、**『エラーが出てしまって原因が分からない』**、**『やりたいことをどうすればできるのかが分からない』** とか」  
りすりす「もちろん **このサイトの内容でわからないことがあったとき** もだけど」  
りすりす「そんな時は **まず、自分で解決をしようと頑張ってみよう**」

どんぐり「人に聞いちゃいけないの？」

りすりす「**自分で調べてもわかりそうにないってなったら、人に聞くべき** だね」  
りすりす「**わからないことをわからないまま放置するのが一番よくない** からね」  
りすりす「Google も、[15 分間自分で調べても分からなかったら人に聞こうって言ってる](https://twitter.com/math_rachel/status/764931533383749632) よ」
りすりす「人に聞くときは、使っている言語など自分のプログラムの事を細かく説明しよう」  
りすりす「調べた事や考えたことも書くべきだね」

どんぐり「調べるって、具体的にどうすればいいの？」

りすりす「**エラーメッセージが出ているのならそれを読んでみよう**」

どんぐり「英語は苦手だよぅ」

りすりす「英語が苦手なら、[Google 翻訳](https://translate.google.com/) などの翻訳ツールを活用してみよう」  
りすりす「エラーメッセージを読んでも原因が分からなかったら、**エラーメッセージを Google 検索をしてみよう**」  
りすりす「**やりたいことやわからないことがある時も、Google 検索をしてみよう**」

どんぐり「Google 検索のコツを教えて」

りすりす「まず、**検索ワードは簡潔に書く** ことだね」  
りすりす「例えば JavaScript で配列の中身を全て繋げた文字列を作る方法が知りたいなら」  
りすりす「`js 配列 結合 文字列` って調べるといいよ」  
りすりす「あとは、**一つのサイトを見ても分からないなら他のサイトも見るべき** だね」

どんぐり「いつも一番上のサイトしか見てなかった......」

りすりす「それと、**英語で調べる** とマイナーなことでも出てきたりするから」  
りすりす「日本語で調べても知りたいことが出てこない！ってなったら英語で調べてみるといいよ」
